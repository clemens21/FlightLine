# Temporary Issues

- 2026-03-16: Contract board fit labels can overstate actual aircraft capability. During repeated backend smoke runs, an offer marked `flyable_with_reposition` (`KSPS -> KLRD`, cargo 1329 lb) was accepted for a Cessna 208 passenger variant, but `CommitAircraftSchedule` then failed with `Cargo weight exceeds the modeled payload capability.` The active cabin layout only exposed 1267 lb cargo capacity, while the board fit logic in `contract-board-generator` appears to classify offers from company phase/footprint rather than the active aircraft cabin payload envelope.
- 2026-03-16: Launcher delete appears unreliable for a save that was just created and opened. In the new Playwright launcher flow, confirming delete on a save immediately after opening it through the shell returned to the launcher but left the save row in place. Deleting a separate unopened save from the same launcher session succeeded, which suggests the delete path may be racing an open handle or file lock on recently opened saves.
- 2026-03-16: Overview data stays stale after contracts-side mutations until the shell reloads. In Playwright coverage, accepting a planned offer on the Contracts tab updated shell chrome and the active-contracts view, but switching back to the Overview tab still showed `No accepted company contracts yet.` until the page was reloaded. The dashboard tab cache is likely not being invalidated when contracts actions mutate company work.
- 2026-03-16: The current workspace source does not compile cleanly, which blocks the normal automated test entrypoints. `npm run build` now fails in `src/application/backend-service.ts` with `TS2345: Argument of type 'RefreshAircraftMarketCommand' is not assignable to parameter of type 'never'.` This appears unrelated to the test-folder changes, but it prevents `npm run test:unit` and `npm run test:ui` from reaching execution until the command typing is fixed.
- 2026-03-16: The clock quick action for a future selected day is currently inconsistent in the shell. In Playwright repro on a clock payload where `Sim to selected morning` rendered enabled for `2026-03-17`, clicking it returned the flash error `Sim to the selected morning is not available for that day.` and reset the clock panel back to the current date. The future-day selection appears to be lost between the rendered button state and the POST handler validation path.
- 2026-03-16: The shell menus do not currently enforce mutual exclusion consistently. In Playwright coverage, opening the settings menu while the clock menu was already open left the clock menu open instead of collapsing it. The shell click/toggle behavior suggests these menus are intended to close each other, but the browser behavior does not match.
