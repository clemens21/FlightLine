# Wireframes Index

## Purpose

This folder contains first-pass low-fidelity wireframes for the FlightLine MVP core loop.

These are operational wireframes, not polished visual mockups.

## Scope Of This Pass

Wireframe order:

1. `01-dashboard.md`
2. `02-contracts.md`
3. `03-dispatch.md`
4. `04-fleet.md`
5. `05-staffing.md`

## Shared Assumptions

- desktop-first viewport: `1440 x 960`
- same data snapshot across every screen
- same global shell across every screen
- low-fidelity layout first, visual styling later
- right-side detail drawers preferred over full-page context switches where possible

## What To Evaluate

Review each wireframe for:

- information hierarchy
- decision clarity
- cross-screen consistency
- alert visibility
- action placement
- whether the layout still works under realistic data density

## Expected Outcome Of This Pass

This pass should tell us:

- which screens feel structurally strong already
- which screens are trying to do too much
- what should be tightened in the design docs before visual polish starts
